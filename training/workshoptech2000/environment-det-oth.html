DLPS Programming Tools and Environment

XEmacs v.21 and Emacs v.19 (details)

  •  TAGS - ease of source navigation, global searching
  •  HTML/SGML Modes - easier viewing of source
  •  Font Lock Mode ("Fontification")  -  program text is displayed in various fonts and colors (cperl-mode)
  •  Perl debugging Mode
    • Interactive perl debugging with debugger in one window and the source in another.
    • Next line to execute is pointed to.
    • Examine and Set perl variables.
    • Set breakpoints and conditional watchpoints.
    • Syntax checking (compilation without execution).
    • Useful to test perl fragments in isolation e.g regular expressions
  • Comint Mode - allows us to run Pat in an XEmacs window.
  • More useful than shell for command recall and searching through results.

CVS and Work Directory Structure (details)

  • CVS is used for source control.
  • GNU tool Built on top of RCS.
  • Allows parallel development.
  • Automatic source merging.
  • Versioning.
  • Tagging for reproducible release construction independent of development stream.
Each programmer has a work directory below a given release directory.

For example:

      /l1/cgi/t/text/p/pfarber

The TextClass CGI programs are coded to support this work directory structure.  In each work directory, we make a symlink to the CGI with our userid in the name:  text-idx-pfarber -> text-idx.  Then when we send the server a URL such as http://dev.umdl.umich.edu/cgi/t/text/p/pfarber/text-idx-pfarber?..., the CGI strips off "-pfarber", turns it into "p/pfarber" and builds a path to that developer's work directory under /l1/lib and is added to the perl @INC.

Similarly, the path to the developer's work directory is included in all HTML links built by the CGI.  So each developer is isolated from outside changes to the source until he updates his work directory with the changes that have been committed to the CVS repository.
