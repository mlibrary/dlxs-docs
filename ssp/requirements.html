SGML Server Program -- Requirements

We will provide for each collection:
  1. a normalized version of the SGML (not applicable for OED)
  2. a built index
  3. region files
  4. perl5 middleware programs and libraries for search and display
  5. filter to HTML
  6. HTML files
Legal steps (in all cases, for each institution involved if for multiple institutions)
  1. For each participating institution, original signed SGML Server Program document to the Sharyn Sivyer at DRDA and a copy to John Price-Wilkin (302 Hatcher North, Ann Arbor, MI 48109-1205).
  2. Copy of OED tape license, if applicable
  3. Letter(s) from Chadwyck-Healey confirming purchase of collections (Stephen Rhind-Tutt can provide)
Technical steps
  1. path to pat50 (or symlinked version)
  2. path to perl5 (or symlinked version)
  3. name of CGI bin and lib directories where you plan to keep the middleware -- both as seen by the web client, and as it exists physically in the filesystem.
  4. designate a place for the Greek GIFs and include that path (as seen by the web client). For example:
    /usr/local/etc/httpd/icons/greek
    but we'll probably be making more GIFs later on, so you may want to use a more generic name.
  5. directory/path for data dictionary, sgml, index, and region files for each collection. There is no advantage to separating index and SGML files, but should you need to, we can support this. For example: 
    	   /txt1/epd/EPD.sgm
	   /idx1/epd/EPD.idx
	   /rgn1/epd/EPD.rgn
    if you like, or 
    	   /ch1/epd/EPD.sgm
		   EPD.idx
		   EPD.rgn
    instead.

We can fit just about everything you'll get on one 5GB 8mm tape (blocking factor of 0), but we can also do lower density 8mm tapes. If you can set up a temporary account on the machine for us (htisetup), we transfer via rcp/ftp and help diagnose configuration problems more effectively.

OED special information

If you're using Paul Pomes's ox2 or something like it with a previous version of OT/PAT, you'll probably want to keep a parallel version of your old OED resources until you can figure out how to have ox2 talk to the new index (it's just differences in naming conventions), or until you can wean users from the old program. At Michigan, we have discontinued our telnet- and non-Web client-based access, as it was no longer being used.

The middleware and data dictionaries are of inconsequential size -- somewhere less than 1500 lines of perl code for each piece of middleware, around 100kb of perl libraries; something like 70-100kb for each data dictionary.
